Class CameraFlightAnimation

Animates its View's @xeokit/viewer!Camera to look at specified objects, boundaries or locations.

Summary

  • Belongs to a @xeokit/viewer!View, and is located at View.cameraFlight
  • Controls the View's @xeokit/viewer!Camera , which is located at View.camera
  • Navigates the Camera to look at a ViewerObject or boundary
  • Navigates the Camera to an explicit position given as eye, look and up vectors
  • Jumps or flies the Camera
  • Smoothly transitions between projections

flyTo

  • flyTo(params?, callback?): void

  • Flies the @xeokit/viewer!Camera to a target.

    • When the target is a boundary, the @xeokit/viewer!Camera will fly towards the target and stop when the target fills most of the canvas.
    • When the target is an explicit @xeokit/viewer!Camera position, given as eye, look and up, then CameraFlightAnimation will interpolate the @xeokit/viewer!Camera to that target and stop there.

    Parameters

      Either a parameters object or a @xeokit/core!Component subtype that has an AABB. Defaults to the Scene, which causes the @xeokit/viewer!Camera to fit the Scene in view.

      • Optional aabb?: FloatArrayParam

        World-space axis-aligned bounding box (AABB) target to fly to.

      • Optional duration?: number

        Flight duration in seconds. Overrides CameraFlightAnimation.duration.

        Position to fly the eye position to.

      • Optional fitFOV?: number

        How much of field-of-view, in degrees, that a target ViewerObject or its AABB should fill the canvas on arrival. Overrides CameraFlightAnimation.fitFOV.

      • Optional length?: number
      • Optional look?: FloatArrayParam

      • Optional orthoScale?: number

        Animate the Camera's orthographic scale to this target value. See Ortho.scale.

      • Optional projection?: number

        Projection type to transition into as we fly. Can be any of the values of Camera.projectionType.

jumpTo

  • jumpTo(params): void

  • Jumps the @xeokit/viewer!View's @xeokit/viewer!Camera to the given target.

    • When the target is a boundary, this CameraFlightAnimation will position the @xeokit/viewer!Camera at where the target fills most of the canvas.
    • When the target is an explicit @xeokit/viewer!Camera position, given as eye, look and up vectors, then this CameraFlightAnimation will jump the @xeokit/viewer!Camera to that target.
